さて、問題を考えます。
「cssを読み込めない」というエラーが発生しているはずなのに、ページには色がついています。
エラーが起きているcssは

https://sbkinoko.github.io/html_css/css_error/style.css

です。

指定した覚えもなければ、作成もしていないのでエラーがでてもしょうがないですね。

開発者ツールでnetworkを確認すると、headerで指定しているcss・jsファイルはどちらも存在しています。
やはり知らないところで勝手に指定されているcss(css_error/style.css)が悪さをしています。


  • 他のページでも同様の問題が起こっていること
  • root直下に置いていて同じフォルダにcssがあるtopページでは問題が起こってないこと

    • から、勝手に同じフォルダにあるcssを指定する問題があると考えます。
    ではなぜ、同フォルダのcssを探しに行ってしまうのでしょうか？


    答えは「stylesheetを指定していないから」でした。
    どうことかというと、もともとは「styleSheet」とSを大文字で指定していました。なので、「stylesheet」が未設定になり、同階層にあるであろうデフォルトのcssを読みに行ってしまったわけです、多分ね。
    改めてcssの設定について書いてあるサイト[1]を見ていたら、気づくことができました。丁寧に見比べるのは大事ですね。
    それはそれとして、指定を間違っているのにcssを読み込んでその通り表示するロジックも気になりますね。いつか気が向いたら調べます。

    ということで今回の教訓は、「大文字と小文字の間違いには気を付けましょう」でした。
    ではまた。
